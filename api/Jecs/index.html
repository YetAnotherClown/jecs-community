Jecs

Just a stupidly fast Entity Component System.

The Repository for Jecs can be found here.

Types​

Entity​

</>
type Entity = number

An alias for Jecs.Id.

Id​

</>
type Id = number

An Id represents an Entity, a Component, or a Pair. In Jecs, it is typed as { __T: T } to preserve types on Entities/Components, though it is essentially just a number.

Pair​

</>
type Pair = number

Represents a relationship, where the first element is the relation, and the second element is the target. You can create pairs with Jecs.pair.

Component​

</>
type Component = number

An alias for Jecs.Id.

Properties​

OnAdd​

</>
Jecs.OnAdd: number

A Component Trait for setting a hook onto a Component, which will run when the Component has been added to an Entity. 

local Transform: Id<vector> = world:component()
world:set(Transform, Jecs.OnAdd, function(entity: Entity, id: Id<T>, data: T)
	-- `Transform` has been added to `entity`
end)

OnRemove​

</>
Jecs.OnRemove: number

A Component Trait for setting a hook onto a Component, which will run when the Component has been removed to an Entity. 

local Transform: Id<vector> = world:component()
world:set(Transform, Jecs.OnRemove, function(entity: Entity, id: Id<T>)
	-- `Transform` has been removed from `entity`
end)

OnChange​

</>
Jecs.OnChange: number

A Component Trait for setting a hook onto a Component, which will run when the data of a Component has been changed on an Entity. 

local Transform: Id<vector> = world:component()
world:set(Transform, Jecs.OnChange, function(entity: Entity, id: Id<T>, data: T)
	-- `Transform` on `entity` has been changed to `data`
end)

Wildcard​

</>
Jecs.Wildcard: number

An Id used for Wildcard Queries.

local Likes = world:component()
local Name = Jecs.Name
local Wildcard = Jecs.Wildcard
for id, name in world:query(Name, pair(Likes, Wildcard)) do
	local nth = 0
	local likes: Entity? = world:target(entity, Likes, nth)
	while likes do
		local friendsName: string? = world:get(likes, Name)
		print(`{name} likes {friendsName}`)
		nth += 1
		likes = world:target(entity, Likes, nth)
	end
end

ChildOf​

</>
Jecs.ChildOf: number

An Id used for Parent-Child hierarchies using Relationships.

local ChildOf = Jecs.ChildOf
local Character = world:component()
local Player = world:component()
for entity in world:query(Character) do
	local player: Entity? = world:parent(entity)
	print(world:has(player, Player)) -- Output: true
end

OnDelete​

</>
Jecs.OnDelete: number

A Cleanup Condition which executes when the Entity is deleted. This is meant to be used with Jecs.Remove and Jecs.Delete to implement Cleanup Traits.

OnDeleteTarget​

</>
Jecs.OnDeleteTarget: number

A Cleanup Condition for Relationships which executes when the target of the relationship is deleted, where the target is the second element of a pair. This is meant to be used with Jecs.Remove and Jecs.Delete to implement Cleanup Traits.

Delete​

</>
Jecs.Delete: number

When used as apart of a Cleanup Trait, it will delete all entities with the relevant Id. 

local OnDelete = Jecs.OnDelete
local Delete = Jecs.Delete
local Component = world:component()
world:add(Component, Jecs.pair(OnDelete, Delete))
local entity = world:entity()
world:add(entity, Component)
world:delete(Component) -- `entity` will be deleted.

Remove​

</>
Jecs.Remove: number

When used as apart of a Cleanup Trait, it will delete all entities with the relevant Id. 

local OnDelete = Jecs.OnDelete
local Remove = Jecs.Remove
local Component = world:component()
world:add(Component, Jecs.pair(OnDelete, Remove))
local entity = world:entity()
world:add(entity, Component)
world:delete(Component) -- `Component` will be removed from `enttiy`

Name​

</>
Jecs.Name: number

An Id used for setting names on entities.

local Name = Jecs.Name
local alice = world:entity()
world:set(alice, Name, "Alice")

Exclusive​

</>
Jecs.Exclusive: number

A Component Trait for marking relationships as exclusive, meaning that the entity can only ever have one target for the relation marked as exclusive, where the target is the second element of a pair.

Functions​

component​

</>
Jecs.component() → Component<T>

Preregisters a component, without a World.

tag​

</>
Jecs.tag() → Component

Preregisters a tag component, without a World.

meta​

</>
Jecs.meta() → ()

Sets metadata onto a Component, without a World.

local Name = Jecs.Name
local Position = Jecs.component()
Jecs.meta(Position, Name, "Position")

pair​

</>
Jecs.pair() → ()

Encodes two Ids into one, to represent a Relationship.

local Likes = world:component()
local alice = world:entity()
local bob = world:entity()
world:add(bob, Jecs.pair(Likes, alice))

For more information on pairs, read here.

world​

</>

Creates a new World.
