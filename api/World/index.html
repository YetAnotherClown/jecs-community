World

A world stores all of your Entities and Components, providing an API to manage them.

Types​

Column​

</>
type Column = {any}

Archetype​

</>
interface Archetype {
id: number
types: {Entity}
type: string
entities: {Entity}
columns: {Column}
columnsMap: {[Entity]: Column}
}

Represents the internal storage of the ECS. For more information on Archetypes, read here.

range​

</>
World:range(
rangeBegin: number,--

The id to start at

rangeEnd: number?--

The optional id to end at

) → ()

Enforces that entities created will have ids between the given range.

world:range(1000, 5000)

local entity = world:entity()
print(entity) -- Will be >= 1000 and < 5000

query​

</>
World:query(
...: Id--

The Components to query for

) → Query--

The Query object

Creates a Query object to iterate over all entities with the given components, returning the entity id and values of each component.

TIP

Queries can do much more! See Query to see the additional methods you can chain together.

for id, health, position in world:query(Health, Position):with(Enemy) do
	-- ...
end

target​

</>
World:target(
entity: Entity,--

The Entity Id

relation: Id,--

The first element of a pair

nth:: number?
) → Id?--

The target, or the second element of a pair

Given a relation, this will return the target of the nth relationship of the entity, where the first element of a pair is the relation and the second element of a pair is the target. 

local ChildOf = Jecs.ChildOf
local Character = world:component()
local Player = world:component()

for entity in world:query(Character) do
	local player: Entity? = world:target(entity, ChildOf)
	print(world:has(player, Player)) -- Output: true
end

local Likes = world:component()
local Name = Jecs.Name
local Wildcard = Jecs.Wildcard

for id, name in world:query(Name, pair(Likes, Wildcard)) do
	local nth = 0
	local likes: Entity? = world:target(entity, Likes, nth)

	while likes do
		local friendsName: string? = world:get(likes, Name)
		print(`{name} likes {friendsName}`)

		nth += 1
		likes = world:target(entity, Likes, nth)
		end
end

parent​

</>
World:parent(
entity: Entity--

The Entity Id

) → Id?

Returns the parent of an entity, where the parent is the target of a ChildOf relationship, or nil if there is no relationship pair. 

local ChildOf = Jecs.ChildOf
local Character = world:component()
local Player = world:component()

for entity in world:query(Character) do
	local player: Entity? = world:parent(entity)
	print(world:has(player, Player)) -- Output: true
end

remove​

</>
World:remove(
entity: Entity,--

The Entity Id

id: Id--

The component to remove

) → ()

Removes a component from the given entity.

delete​

</>
World:delete(
entity: Entity--

The Entity Id

) → ()

Deletes an entity, as well as all of it's components and relationships.

clear​

</>
World:clear(
entity: Entity--

The Entity Id

) → ()

Clears all of the components and relationships from the given entity, but does not delete the entity. Instead, you can use World:delete to delete an entity and all of it's components.

exists​

</>
World:exists(
entity: Entity--

The Entity Id

) → boolean

Given an entity id, returns true if the id exists in the world, or false, disregarding whether or not the entity is alive.

contains​

</>
World:contains(
entity: Entity--

The Entity Id

) → boolean

Given an entity id, returns true if the id exists and is alive in the world, or false.

cleanup​

</>
World:cleanup() → ()

Cleans the world of empty archetypes.

The interval at which you should call this method varies per application, and you should profile your application to determine this interval.

component​

</>
World:component() → Component

Creates a new component.

TIP

Components are also entities in Jecs, where the first 256 entity ids are reserved for components.
